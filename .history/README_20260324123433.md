# STEP 2

install any initial libraries

- express - node framework (makes creating a node server easier, like React for frontend)
- cors - let's different origins access your backend
  - example: React applications run on http://localhost:5173 => trying to access your backend http://localhost:3000
  - These are different ORIGINS
  - CORS allow us to configure permission settings for the browser
    - 1. Allowed Origins { origin: 'http://localhost:5173' }
    - 2. Methods: ['GET', 'POST']
    - 3. Allowed Headers: ['Content-Type', 'Authorization', Etc]
    - 4. Credentials: true / false (allows for cookies, sessions, auth tokens, etc)
- dotenv - access variables from our .env file.
  - .env hold all of our secret variables or variables we don't want to expose to the client/public/repo/etc
- typescript - npm install typescript --save-dev
  - Additional Packages: npm i -D @types/node @types/express tsx
    - types/node: provides typescript type definitions for node.js core modules (http, path, fs, etc)
    - @types/express: gives us type checking for express. ie: request, response, nextfunction 
    - tsx: Typescript execution envvironment: kind of replaces node as your environment that your code rusn in 
      has some built in stuff like: hot reloading and is replacement for older packages that we had to combine
- prisma (ORM) - helps you do sql, gives you methods/functions that make doing sql easier. Also has a visualizer for your tables
  (better than some of the other options out there, pgAdmin, DBeaver)
    - installing prisma | npm i prisma -D 
    - installing prisma client | npm i @prisma/client
    - [npx prisma studio] this'll host the visualizer on a port for you

    1. Run npm prisma init (initializes prisma in your application, creates a prisma folder)
    2. Look into your new prisma config. On line 12 you'll see that you'll need to update your environment variable. This is the connection string that let's us connect to our docker postgres image
      - STRUCTURE ["postgresql://USER:PASSWORD@localhost:PORT/DATABASE?schema=public"]

# STEP 2.5 TYPESCRIPT

setting up typescript - Read the docs if you want to know more about what each setting does: [https://www.typescriptlang.org/docs/]
- Run npx tsc --init to initialize your typescsript config file. This'll create this file tsconfig.json
** REPLACE THE ENTIRE THING IN THERE WITH THIS, THIS SETUP WILL LET YOU DO MODULE (IMPORT / EXPORT) IN YOUR PROJECT**
[tsconfig.json]
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./dist",
    "module": "CommonJS",
    "moduleResolution": "Node",
    "target": "ES2022",
    "types": ["node"],
    "sourceMap": true,
    "strict": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "skipLibCheck": true,
    "moduleDetection": "force"
  },
  "include": ["src"]
}

In your package.json add in these script files
[package.json]
- "dev": "tsx watch src/server.js"

This will be the one we're running ( replaces our node --watch server.js)
